## Overview

- Flink is a scalable, fault-tolerant distributed system
- Flink uses SQL syntax for stream processing
- Flink aggregation need to keep states
- Flink states are regularly written into a remote distributed file system
- Flink uses snapshots to keep track of the local state throughout the distributed systems
- Flink uses watermarks to do windowing, to determine how long should it wait for a event data
